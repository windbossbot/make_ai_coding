# Operations Guide

## 실행 전 점검
- `.env` 파일을 만들고 민감 정보는 환경변수로만 주입합니다.
- 장기 실행 모드에서는 시작 전에 포트 충돌을 확인합니다.
- PID 파일이 남아 있으면 실행이 중단되며 정리 명령어가 함께 제시됩니다.

## 안전 장치
- 파괴적 작업은 `dry_run`이 기본입니다.
- 실제 쓰기, 삭제, 주문, 출금 류 작업은 `--force` 또는 명시적 승인 없이는 진행하지 않습니다.
- 외부 입력값은 즉시 검증하고 비정상 범위는 예외로 중단합니다.

## 로그
- 로그는 `logs/YYYY-MM-DD.log`에 JSON 형태로 저장됩니다.
- 최소 운영 레벨은 `INFO`, 개발 시 `DEBUG`를 권장합니다.

## 규제 및 윤리
- 금융/개인정보 기능을 추가할 때는 관할 규제 요건을 코드 주석 또는 설계 문서에 남기세요.
- 시장조작으로 해석될 수 있는 전략은 금지 규칙과 함께 구현하세요.
